HD Lands development guide
Это новый РЕбилд мода, пересборка с чисткой хлама изменением структуры и пр.

Содержание:
1. --
2. Описание структуры мода
3. Структура мода
4. О системе сборки
5. О системе компиляции

1. Запрет на использование (пункт утратил силу с 31.01.2024)
1.1 для некоторых модов: Jamais Vu, Revenge.
1.2 для некоторых модмейкеров (и всех их модов): Перун Иглес, Алексей Чёрный.
1.3 для определённых пользователей: null.

2. Новая структура:
2.1 Каждая карта пакуется в свой .res - спокойно можно подключать/отключать модмйкерам определённые карты.
2.2 Ресурсы делятся на Модели(префикс "m_") и Текстуры("t_"), а потом на Природу, Конструкции и Юнитов.

3. Итоговая структура мода:
config.reg - конфиг HD Lands
readme.txt - описание HD Lands
dev.txt - описание для модмейкеров (этот документ)
*.bat - скрипты системы сборки
HDLandsSpecific/ - ресурсы, специфичные для HD Lands
  - hdlands - версия игры в главменю; титры, объязательны для включения их содержания в титры вашего мода!
  - movie.ini - настройка загрузочного видеоролика
HDRes/ - смешанные ресурсы (модели и текстуры)
  - share - Модели и Текстуры общего назначения + новые ресурсы
HDModelsRes/ - модели большего поликаунта
  - m_nature - Природа
  - m_objects - Конструкции
  - m_units - Юниты
HDTexRes/ - текстуры большего разрешения
  - t_nature - Природа
  - t_objects - Конструкции
  - t_units - Юниты
  - t_world - Общие элементы для всех карт игры (магия, принты)
  - t_zone{0} - Зона "{0}"
tools/ - инструменты
url/ - папка для голубей и ссылок

4. система сборки (*.bat)
набор .bat файлов в корне мода и папка tools там же - позволяет проводить различные манипуляции с .res файлами, ускоряя работу. Из за особенностей системы сборки, в пути до мода не должно быть пробелов, из за чего в названии папки они и заменены на символ _

5. Система компиляции атласов (HDL_buildsystem.7z)
Доступна на Яндекс.Диске модификации, позволяет получать набор .mmp текстур террейна из базовых атласов.
