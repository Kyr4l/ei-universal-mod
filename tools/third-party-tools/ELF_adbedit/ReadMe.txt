Примечания к приложению:

ВНИМАНИЕ: Примечания не проверены, поэтому могут не совпадать с реальностью :)

• Название модели:
	Это название должно совпадать с названием самой графической модели в ПЗ. Оно как раз является идентификатором того, для какой модели будет использоваться данный ADB в качестве условия. Максимальная длина названия - 23 буквы.

• Максимальная высота модели:
	Максимально допустимая высота модели, для которой используется данный ADB. Например, если Зака сделать огромным, то Зак начнёт топать, как людоед. Это связано с тем, что ADB для простого человека к огромному Заку уже не подошёл, и игра нашла более подходящий ADB - в этом случае от людоеда.

• Средняя высота модели:
	Скорее всего это рекомендуемая высота модели. Например, если сделать два схожих ADB файла, в одном сделать рекомендуемую 0.5, а в другом 0.9, то для модели с ростом 0.5 будет играться первая, а для модели с ростом 0.9 вторая, не смотря на то, что пределы у обоих моделей будут одинаковые (Максимальная и Минимальная высоты).

• Минимальная высота модели:
	Минимальная высота модели для которой используется данный ADB файл.

• Список действий модели:
	В этот список вносятся при загрузке все действия загружаемой модели. Можно добавлять новые действия ("Добавить действие") или удалять имеющиеся ("Удалить действие"). Две мрачные кнопки справа от списка позволяют перемещать действия по списку. Вообще вроде бы расположение действий в списке никакой роли не играют, но для удобства эти функции присутствуют.

• Название текущего действия:
	Название действия. Меняется оно синхронно с названием в списке. Название может быть не больше 15 букв.

• Личный номерок действия:
	Это ID действия. Скорее всего игра ориентируется в действиях по этому номерку.

• Допустимые оружия:
	Здесь указываются допустимые оружия. Это значит, что для проигрывания данного действия нужно, чтобы у модели было активировано (в руках находилось) хотя бы одно из перечисленных оружий. Оружия можно указать галочками, можно указать значение оружий, а можно ввести в результат список оружий, вот то, что можно вводить в результат:
	• SWORD - Меч;
	• AXE - Топор;
	• DAGGER - Кинжал;
	• SPEAR - Копьё;
	• HAMMER - Молот;
	• BOW - Лук;
	• CROSSBOW - Арбалет;
	• ALL - Всё, или всё остальное.
	Эти параметры нужно вводить через запятую, например: "SWORD, BOW".

• Этап анимации:
	Скорее всего предназначен для тех случаев, когда нужно для проигрывания действия использовать несколько анимаций поэтапно. Например, бег человека - человек начинает бежать из состояния стоя - это первое действие; потом человек бежит - это цикличное действие, которое идёт после первого и будет идти до конца бега; а затем человек останавливается - и это уже будет третье действие, показывающее переход от бега к ходу. В этом разделе результат может иметь только 4 значения, причём можно указать только ОДНО из четырёх значений. Вот то, что можно ввести:
	• UNIQUE - Уникальное состояние, которое не подходит к фазовым действиям;
	• START - Начальное действие;
	• CYCLE - Циклическое действие;
	• END - Конечное действие.

• Вероятность действия:
	Вероятность того, что проиграется именно данное действие. Этот параметр скорее всего предназначен для тех случаев, когда для одного реального действия предназначено сразу несколько анимаций - тогда идёт выбор любой из них с учётом вероятности. Вероятность указывается от 1 до 100 (в процентах).

• Длина анимации (во фреймах):
	Длина анимации. Один фрейм длится примерно 1/15 секунды.

• Скорость передвижения:
	Скорость, с которой модель должна двигаться при проигрывании действия. Где не нужно двигаться, там этот параметр имеет нулевое значение.

• Начальный фрейм удара:
	Точно не знаю, может быть обозначает, на каком фрейме передаётся удар от данной модели к модели-жертве.

• Допустимые состояния:
	Состояние модели, при котором допускается проигрывание данного действия. Вот варианты результата:
	• NEUTRAL - Фактически говорит о том, что состояние не играет роли;
	• REST - Только для состояния отдыха;
	• ATTACK - Только для состояния атаки;
	• WARRY - Только для настороженного состояния;
	• LIE - Только для положения лёжа;
	• ALL - Для всех указанный состояний.

• Тип действия:
	Тип, к которому относится данное действие. Тип действия также зависит от модификатора типа (этакого подтипа). Набор модификаторов у каждого типа действия свой. Вот обзор типов:
	• ATTACK - Действие относится к типу "атака";
	• CAST - Действие относится к типу "колдунство";
	• RUN - Действие относится к типу "бег";
	• WALK - Действие относится к типу "ходьба";
	• IDLE - Действие относится к типу "стоять";
	• DEATH - Действие относится к типу "смерть";
	• SUFFER - Действие относится к типу "пропуск удара";
	• CROSS - Так и не понял, к чему собственно это относится;
	• ALL - Действие относится ко всем типам;
	• SPECIAL - Дейтсвие имеет уникальный тип.

• Наличие в действии:
	Что присутствует в действии. В результат вносится список через запятую. Вот список допустимых значений:
	• HIDES - Сокрытия;
	• STEPS - Шаги;
	• HIT - Удар;
	• SFXES - Звуки;
	• LOWSHAPE - Низкоуровневая форма;
	• HIGHSHAPE - Высокоуровневая форма.
	Вот пример: "HIDES, LOWSHAPE".

• Начальный фрейм для показа/сокрытия 1/2:
	Скорее всего предназначен для моделей-снарядов. :)

• Начальный фрейм для озвучки шага 1..4:
	Скорее всего фрейм, начиная с которого будет идти озвучка шага. А 1..4 - варианты.

• Начальный фрейм для особой озвучки:
	Начальный фрейм для особой озвучки, вроде используется для спец.звуков.

• Имя спец.звука 1..4:
	Имя спец звука. Причём, для 156 будет соответствовать имя "156.wav" и т.п.


• Скопировать действие:
	Копирует действие во внутренний буффер обмена.

• Вставить действие:
	Вставляет скопированное действие из внутреннего буффера обмена вместо текущего.

Можно скопировать действие из одного ADB-файла и вставить в другой. ;)
