List of script functions for the game "Evil Islands"

Function Description
ActivateTrap (
Trap: object, Activate: float)		 Command: ActivateTrap - deactivates: Activate - 0 and activates: Activate<>0 the trap: Trap on the game zone. For example, on the game zone of the map: Foothills, you should enter in the console: @ActivateTrap( GetObjectByID( "4515" ) , 1 ) will activate the pyramid of the Ork tribe. This trap-Pyramid attacks only Diplomatic group - 0. It is impossible to reconfigure diplomacy for Trap objects using the command: SetDiplomacy.

AddLoot (
nPlayer: float, PartyNameSend: string, PartyNameRecv: string)		 Command: AddLoot - adds items and money from the 'Party' train: PartyNameSend to the train of NOT the current 'Party': PartyNameRecv. Both 'Parties' belong to the 'Player': nPlayer. Do not confuse 'Train' and 'Backpack'. In case if changing the current party will occur on the game zone, then items from the 'Backpack' set always included in the 'Train' set will also be copied to the train of 'Party': PartyNameRecv. When executing the command: RedeployParty 'Party Backpack': PartyNameSend will be zeroed. The 'Backpack' of the new party will also be empty, although items will remain in the backpack info window, but in fact the backpack of the previous and current Parties will be empty. Function: GetLootItemsCount will return value 0.

AddRectToArea (
idArea: float, x1: float, y1: float, x2: float, y2: float)		 Command: AddRectToArea - adds the belonging of a game zone section to the quest area under the code: idArea. The area is described as a rectangle, where (X1,Y1) and (X2,Y2) are the coordinates of its two opposite corners. (Left-bottom and top-right) (south-west and north-east corner). The sides of the rectangle are always located parallel to the cardinal directions. If the coordinates of the second point are less than the first, there will be no zone. There can be many areas with the same code: idArea on the game zone. For example, the command: QObjArea can set the task for the 'Player' (Player team): to get into the area under the code: idArea - as one of the conditions for completing the quest declared by the command: QStart. In this case, the 'Player' must get into any of the sections of the area with the code: idArea.

For example, the command: QObjArea can set the task for the 'Player': to get into the area under the code: idArea as one of the conditions for completing the quest declared by the command: QStart - in this case the 'Player' must get into any of the sections of the area with the code: idArea.

AlarmTime (
nAlarm: float): float		 Function: AlarmTime - returns the value of the variable: nAlarm. When activating the variable: nAlarm by the command: InvokeAlarm, it is assigned a starting value. The starting value is the sum of the amount of time of all time intervals, between activations and deactivations of all previous variables: nAlarm from the start of the game. Only one number variable can be active. To the variable: nAlarm after activation, the value of the time counter (1/15 sec) is constantly added. Activating the next number of the variable: nAlarm by the command: InvokeAlarm simultaneously deactivates the previous variable, but does not zero it. The variable number does not necessarily have to be in order and you can reinitialize the same number variable. To reset all values of variables: nAlarm to zero and stop the counter in the global variable, you need to call the command: InvokeAlarm with the first parameter = 0 : InvokeAlarm(0,0,0)

CreateLightning (
id: float, x1: float, y1: float, z1: float, x2: float, y2: float, z2: float, Intensity: float)		 Command: CreateLightning - creates a constantly acting frozen lightning, striking from point: (x1,y1,z1) to (x2,y2,z2) with intensity: Intensity. With a positive value: Intensity point: (x2,y2,z2) - represents a ball - as a result of a lightning strike on an object. With a negative value: Intensity the lightning represents a frozen lightning of natural origin. With the value: Intensity = 0 the lightning represents a type of spell: Barrier of lightning with intensity (thickness of discharges) by default. The intensity value: Intensity different from zero, is graphically represented as the thickness of discharges. It is not advisable to set the value Intensity greater than 20, since this strongly slows down the game on the game zone. If you set the value: Intensity = 100, you will get something like the effect of the northern lights. Repeated creation of lightning by the command: CreateLightning with the same code: id does not delete the lightning. Thus, you can create many lightnings on the game zone with the same code: id. The coordinates of points: (x1,y1,z1) and (x2,y2,z2) in the command: CreateLightning are in WCS and do not depend on the surface and relief of the game zone, this should be taken into account when setting the value of coordinates: z1 and z2. Delete all subsets of lightnings with the same code can be done by the command: DeleteLightning.

PlayAnimation (
Unit: object, Animation: string)		 Command: PlayAnimation - plays an animation - movement of body parts and accompanying sound effects. Unit - the unit that will play the animation. Where to get anim: First you need to decide on the unit that needs to be set in motion. Then in lmpdatabase\MapObjects\RaceModels.txt find the Name of this unit. Now look at what is in the Mask name column. We need a file with such a name and extension adb, it is located in res\database.res\ . Open this file with notepad and see among the mess of icons some coherent elements - these are the anims of this type of unit. I checked on the example of an orc-shaman: Name = LMP Orc Male Tatoo Mask name = unorma File = res\database.res\unorma.adb The anims dug out from there: uattack01..09 - Attack with various weapons. Falls forward / backward / to the side, while running / walking / crawling, with agony / sharply / slowly. cidle01..09 - Looking around, shifting from foot to foot. What the character does when standing in place for a long time. ucast01..04 - Spellcasting. Effects like an elemental ball do not appear, only body movement. uhit01..21 - Hit on the unit. Also like death, from any positions. crun01..02 - Running. Tracks appear under the feet. cwalk01..09 - The unit walks, sneaks. There are animations like a wounded gait, tired, lame. ccrawl01 - The unit crawls. ucross01..09 - Transitions between standing / sneaking / lying position of the unit. scross01..09 - One phase of walking / crawling etc. Full step, or one dash to the side. crest - The unit sits down to rest. Besides them, in the 'human' file unhuma.adb I found: (did not check) ubriefing## - Movements during dialogues. uspecial## - Salutes, bows, stands at attention, etc. cspecial## - Some activities like sharpening a sword, sitting on a log, etc. The first letters of the animation are indicators: u - the action will happen once, then the unit will return to the initial state c - the action repeats continuously, the unit does not react to the indication of another animation. If at this time you hit the unit or appear in sight (of a hostile), he will not react. If you beat a frozen animation object, it will flinch and then again perform the animation specified by the command. If you kill, it will die as usual and lie down as a corpse. s - similar to u. When entering an incorrect anim, nothing happens. If the unit was moving during the procedure, then it will stop, play the animation and then continue.
